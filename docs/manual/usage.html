Usage & Commands

Master the PPMAP CLI with advanced scanning techniques, pipeline integrations, and enterprise configurations.

Basic Scanning

Run a full automated assessment using the 32 detection methods.

# 1. Full Assessment (Discovery + Scan)
$ python3 ppmap.py --scan-full "https://example.com"
# 2. Targeted Scan (Provided URL only)
$ python3 ppmap.py --scan "https://example.com/api/v1"
# 3. Discovery Only (Recon Mode)
$ python3 ppmap.py --discover "https://example.com" --max-depth 2

Enterprise Scenarios

PPMAP is built for scale. Here are real-world deployment examples.

1. CI/CD Pipeline Integration (Mass Scanning)

Pipe targets directly from your recon tools (e.g., Subfinder + Httpx) into PPMAP using --stdin.

$ subfinder -d example.com | httpx | python3 ppmap.py --scan --stdin
# Alternatively, read from a file
$ python3 ppmap.py --scan -ls targets.txt --workers 10

2. Authenticated Scans (Burp Suite Integration)

Export a raw HTTP request from Burp Suite ensuring all Cookies and Auth Headers are included.

$ python3 ppmap.py --request req.txt --stealth

3. Out-of-Band (OOB) Blind Detection

Automatically deploy and monitor an Interact.sh payload for asynchronous/blind vulnerabilities.

$ python3 ppmap.py --scan "https://api.example.com" --oob

Advanced Flags Reference

Flag Description
--scan-full URL Run Discovery + Deduplication + Deep Scan (Standard behavior).
--discover URL Crawl target for endpoints & parameters without launching attacks.
--scan URL Perform targeted scan on provided URLs. Skips discovery phase.
--max-depth N Set maximum crawling depth (default: 3).

💡 Pro Tips

  • 🎯 Targeting: Always use --poc first to check for basic jQuery vulnerabilities before running a deep --scan.
  • 🕵️ Stealth: When scanning strict WAFs, combine --stealth with --delay 2 to lower noise.
  • 📊 Reporting: Use --format json,html and --template modern for executive-ready deliverables.
